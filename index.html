fReady by matt holden

Kick-start your Facebook Platform development!

fReady (or, "Freddy") is a light wrapper to Facebook's Javascript SDK, with zero dependencies. Write less code, and have more fun!

This project is in its earliest infancy, and the documentation is far from complete. If you think it's cool, feel free to contribute.

Install

  • Add fReady.js to your project, then add this line in your <body>
    <script type="text/javascript" src="/path/to/fReady.js"></script>
  • That's it!

fReady will create the fb-root element and download the Javascript SDK for you. (Choosing a specific locale is not yet supported.) However, you can still load the SDK synchronously and include the fb-root element if you prefer -- fReady will abide.

Usage

You can still use the Javascript SDK! All FB.* methods still work.

Start by calling fReady.init()

// Add reference to fReady
<script type="text/javascript" src="/path/to/fReady.js"></script>

// Initialze Facebook with your appID.
fReady.init({
  appId  : YOUR APP ID HERE
  //Pass other FB.init() properties here if you like
  //otherwise, defaults are assumed
});

Next, pass fReady() one or more functions to execute after the user's login status is determined.

This example checks if current user is connected to your app, and alerts their name.
Try it out, tell me my name.

fReady(function(){
  if (fReady.isConnected()){
    var your_name = fReady.getMe().name;
    alert(your_name) //User's name.  ()
  }
});

You can also pass fReady() specific functions to execute after the user's status is determined. fReady can also store functions to execute when the user's status changes. You may omit any or all of these functions. The onStatusChange functions are especially useful if you are concerned about the user logging in or out while interacting with your app: 

fReady({
  onGetStatus:{
    CONNECTED: function(response){},
    NOT_AUTHORIZED: function(response){},
    UNKNOWN: function(response){}
  },
  onStatusChange:{
    CONNECTED: function(response){},
    NOT_AUTHORIZED: function(response){},
    UNKNOWN: function(response){}
  }
});

fReady offers some accessor methods to the data it has cached (user object, login status and authorized permissions). This data is accessible synchronously (cached) or asynchronously (fReady will re-fetch the data from Facebook). This is where fReady becomes so coder-friendly -- (fewer callbacks == more fun) == true.

// Use the cached information:
fReady.getMe() // User object;
fReady.isConnected() //is the user connected to your app?
fReady.getAuthdScope() //object specifying which permissions your user has authorized
 
// Ask fReady to double-check the values with Facebook:
fReady.getMe( function( me ) {} );
fReady.isConnected( function( bool ) {} );
fReady.getAuthdScope( function( authdScope ) {} );
